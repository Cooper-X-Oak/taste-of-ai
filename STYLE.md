# STYLE.md — 语言风格基准

> 《舌尖上的AI》全栏目语言风格的唯一基准(single source of truth)。
> 目的:保住栏目身份(舌尖壳 + 从夯到拉评级),同时切掉过度的 AI 腔。
> 适用:站内页面、公众号、小红书。新增内容前读这页;改稿对照这页。

核心原则一句话:**三套语言各管一段,别串味,正文说人话。**

栏目有三套语言,各自保持轻、互不串味:
- **壳 = 烹饪(轻调味)**:就《舌尖上的AI》+ 试吃 + 主播/后厨 + 锐评。够了,不再造菜名。
- **评级 = 从夯到拉(网络梗真词)**:夯 / 顶级 / 人上人 / NPC / 拉。网感来自这里,跟烹饪是两回事。
- **正文 = 实测白话**:prompt / 轮次 / 失败 / token,直说,零隐喻。

"过度 AI 味"的根不是某个词坏,是三套味全堆进同一句(病灶:"夯爆 · 招牌硬菜" = emoji+网络梗+菜名三层焊死)。

---

## 1. 隐喻预算(三层)

把"舌尖 / 试吃 / 从夯到拉"这几套词当**预算**花,不是每句都撒。

### 保留层 — 身份标签,每处只用一次
低成本扛身份的名词标签,照常用,但同一页面别重复:

- 栏目名:《舌尖上的AI》(烹饪壳)
- 活动:试吃(烹饪壳)
- 评级阶梯(真·从夯到拉,网络梗):**S 夯 · A 顶级 · B 人上人 · C NPC · D 拉(完了)**
- 评测格式:横评(同类排进 Tier List)/ 竖评(单个深扒)
- 导航榜单:避雷红榜(只收 D 拉)/ 从夯到拉总榜
- 角色:导演 / 主播 / 试吃员 / 后厨

> 烹饪词到此为止 —— 不再自创菜名(招牌硬菜/家常好菜/预制菜/黑暗料理 这类已删)。
> 段位用真梗词,不要在段位后再焊一个菜名。
> "照骗 / 名不副实"(菜单 vs 实物反差)是文章里的点评,不是段位。
> 段位定义的单一来源 = `docs/.vitepress/theme/tiers.ts`(改梯队只改这里)。

### 弱化层 — 章节标题,去重 + 控 emoji
隐喻可保留一处做记忆点,但不要每个 heading 都泡在隐喻里。emoji 每段最多一个。

| 现状(过度) | 改成 |
|---|---|
| `🐣 小白试吃姿势` / `🎓 专家试吃姿势` / `⭐ 最佳试吃案例` | `小白上手` / `专家上手` / `最佳用法`(全段保留一次"试吃"即可) |
| `菜单(官方描述)` | `官方描述` |
| `🍜 总评:好不好用` | `总评` |

### 禁用层 — 证据正文 + 分析,不许有隐喻
试吃记录、表现分析、锐评理由里只写实话:具体 prompt、轮次、失败点、关键输出、time/token 感受。
段位只在最后那一下用真梗词(夯 / 顶级 / 人上人 / NPC / 拉),不在后面再焊菜名。

## 2. AI 腔黑名单(见到就改写)

- 排比 / 对仗钩子:`谁真能帮上忙,谁是照骗`、`小白姿势谁接得住、专家姿势谁上限高`
- 双重隐喻叠用:`照骗(预制菜🥱C)`、`拉胯(黑暗料理🤮D)` —— 一个词 + 一个隐喻 + 一个 emoji,三重
- 套话:`悬念钩子`、`真香警告`、`不仅…而且`、`一文读懂`、`手把手`
- emoji 雪崩:每个标题都挂、一句多个
- 感叹号连发
- 疑问句钓人(`你还在为 X 烦恼吗`)

---

## 3. 正向写法(VOICE PROFILE 摘要)

- 短句为主,一句一个意思;允许句子片段(`稳,值得常备`)。
- 高压缩:结论先行,理由跟一句。
- 敢下判断,但判断贴证据(轮次 / 失败 / token 感受)。
- 具体名词、机制、数字、踩坑实录,代替形容词。
- 括号用于补一句限定或吐槽,不放抒情。
- 英文术语(skill / token / prompt)保留原样,不硬翻。

正样本基准(语气对标):
> "夯,装了回不去。" / "顶级,值得常备。" / "NPC,可有可无。" / "拉完了,避雷别碰。"

短、准、段位用真梗词、不堆砌 —— 这就是目标语域。

## 4. 改写示例(取自现有真实文本)

**钩子**
- ✗ `同样想查点东西,这几个调研 skill 谁真能帮上忙,谁是照骗。`
- ✓ `同样一个调研任务,这几个 skill 谁能交活,谁在划水。`

**横评引导**
- ✗ `小白姿势谁接得住、专家姿势谁上限高 —— 主播写。`
- ✓ `模糊提示词谁还能用,清晰提示词谁上限高。结论主播写。`

**避雷榜引导**
- ✗ `照骗(预制菜 🥱C)与拉胯(黑暗料理 🤮D)合辑`  ← 菜名叠梗,还错把 C 算进避雷
- ✓ `避雷榜:实测定级「拉」的全在这,踩一个少一个。`

## 5. 落地方式(全站统一,不靠手抄)

1. `docs/skills/*/index.md` 是同一套骨架 —— 改骨架模板 + `scripts/new-skill.mjs` 模板,新页自动按本规则生成。
2. 回刷三个 index(`by-skill` / `by-tier` / `avoid`)和季页 heading。
3. 新 skill 一律走 `new-skill.mjs`,骨架已统一,自然一致。
4. 写成稿(公众号 / 小红书)前,先读本页;改稿对照黑名单逐条过。
