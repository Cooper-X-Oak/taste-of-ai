# 节目宝典

> 《舌尖上的AI》—— 一档「硬核试吃 + 从夯到拉锐评」的 AI 能力测评手册。
> 主厨只认入口实测,不认菜单吹嘘。

## 试吃对象

Claude Code 生态里一切「能力单元」都是食材:
`skill` · `群组skill(plugin)` · `slash 命令` · `subagent` · `workflow` · `harness & loop` · `MCP`。

第一季专测**独立 skill**;群组、工作流、harness 留作后续赛季。

## 两种机制

- **横评 · 擂台赛**(主菜):同品类多选手,**同一道标准考题**,六维同尺打分,排出从夯到拉梯队。
- **竖评 · 主厨特制**(旗舰):单个重磅选手多场景深扒,讲透脾气。

## 护城河:贝叶斯实证

绝不抄菜单。每道菜必有 **L1 实测证据**(prompt / 截图 / 产出)才能下结论。
评分卡专设「**菜单 vs 实物**」一栏 —— 宣传(L3 先验)对照入口实测(L1 后验)。

## 角色分工

- **导演 / 主播 / 试吃员 = 作者本人**:定方向、亲自品鉴、下最终锐评。「毒舌主厨」是作者人设。
- **栏目策划 / 后厨 = AI 助手**:搭台、出题、真跑留证、出初稿,不替主播下口味判断。

## 交付形态

本手册即最终产物:GitHub 上一个带导航、可分享的 VitePress 站点。

- **每个 skill 一篇独立文章**(`skills/<name>/index.md`)+ 一个资料区(`notes.md`,边测边记)。
- **擂台页是综述**:横向对比 + 排名,把流量分发到各 skill 文章。
- 社交平台(公众号 / 小红书)把这些文章一篇篇 drip 出去。
