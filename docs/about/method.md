# 评测方法论

**立场**:就是一个普通用户的个人体验 —— 好不好用、值不值得装。
不做科学评测,不追公信力,个人意见,仅供参考。

每个 skill 用三种「试吃姿势」过一遍:

## 小白上手

给一个模糊、不会写的提示词(普通人随口一说那种),看 skill 能不能接住、自己补全、给出能用的东西。

> 重点:好 skill 在你不会提需求时也能帮上忙,把地板抬高。
> 只有喂好料才出活的,就没那么"好用"。

## 专家上手

给清晰、有思路的提示词,看它的上限能到哪。

## 最佳用法

找出这个 skill **最适合的用法 / 场景** —— 它真正发光的地方。
给读者"原来该这么用"的干货,也是每期的故事高潮。

## 横评里怎么用

同擂台多个 skill:小白姿势、专家姿势都用**同一句提示词**跑一遍做对比,再各自给出最佳案例。

## 留证(个人意见也得有据)

prompt、产出、截图存 `evidence/`,不靠记忆瞎说 —— 别把"我以为"当结论。

## 结论怎么下

一句话说好不好用 → 归档 [从夯到拉](/about/rating) → 谁该装、谁别碰。
